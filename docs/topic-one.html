The History of HTTP

HTTP is how the server and the information on the Internet communicate, allowing users to share and interact with it. Berners-Lee outlined HTTP to have certain functions such as finding an index search in a “hypertext archive”, redirecting a client to another server, “format negotiation”, and operating the file transfer (O'Reilly Media, 2015).


Some key features of HTTP 0.9 were the “client-server” protocol, ASCII protocol, plain text request and response from hypertext, HTML, and documents (O'Reilly Media, 2015).


From 1991 to 1995, HTTP/1.0 introduced new features like multiple sets of header fields, responses that do not have to include hypertext, and responses that can now start with a status line (O'Reilly Media, 2015).


HTTP/1.1 was released publicly in 1997; it offered features such as improvements on previous contents and focused more on performance optimizations like “byte-range requests” and “caching mechanisms” (O'Reilly Media, 2015).


HTTP/2 was soon introduced in 2012, and there was an update to the transport performance (O'Reilly Media, 2015).


The current update is HTTP/3, and a key factor of this update is that it runs on QUIC (Quick UDP Internet Connections), which will help the consistency and speed of a network (GeeksforGeeks, 2025). Overall, it took a lot of dedication and improvement, which led to the current HTTP operating efficiently.

Key Terms to Note

Term Description
HTTP The system that lets computers and websites talk to each other on
the internet.
Client-Server Protocol A way of communicating where the client (like your computer) asks
for something, and the server (the website) gives it back.
ASCII American Standard Code for Information Interchange, a character
encoding standard for electronic communication.
Byte-Range Requests A way for a computer to ask for just part of a file instead
of downloading the whole thing.
HTML Hyper Text Markup Language, the computer code that defines the
structure of web pages.
QUIC Quick UDP Internet Connections, A new system that makes internet
connections faster and more stable, used in the latest version of HTTP.
